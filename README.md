# CARE ID PCF·LCA 데이터 통합 수집 설문

GitHub Pages로 배포되는 정적 HTML 설문 폼입니다.  
로그인은 기관별 비밀번호(GAS 서버 인증), 제출 데이터는 Google Sheets로 수집합니다.

설문은 **DPP 사전 준비 설문**과 **KOTITI LCA(LCI) 데이터** 두 개의 탭으로 나뉘며, 탭별로 완료/미완료 상태와
임시 저장이 독립적으로 관리됩니다. 기관 목록·트랙 매핑·비밀번호는 전부 서버(Code.gs)에만 보관되고
클라이언트에는 노출되지 않습니다.

---

## 구조

```
index.html          ← 설문 폼 (GitHub Pages 진입점)
.nojekyll           ← Jekyll 처리 비활성화
apps-script/
  Code.gs           ← Google Apps Script 백엔드 코드 (git 추적 제외 — GAS 편집기에서만 관리)
```

## 화면 구성

1. **비밀번호 입력** → 서버가 비밀번호를 소속 기관(또는 Master 권한)으로 변환해 반환.
2. **작성자 정보 입력** → 이름·부서·직위·연락처·이메일.
3. **설문 입력 (탭 2개)**
   - **DPP 사전 준비 설문**: 소속 기관에 배정된 공정(원사·원단·염색·봉제 등) 문항.
   - **KOTITI LCA 데이터**: LCI 투입/산출 데이터를 엑셀과 유사한 인라인 표(공정 설명·투입물·에너지·유지보수·산출물)로 입력. KOTITI LCA 대상 6개 제조 기관에만 표시됩니다.
   - 각 탭은 완료/미완료 배지와 진행률을 독립적으로 표시하며, 입력 중 2초 디바운스로 **탭별 임시 저장**(브라우저 localStorage)됩니다. 응답은 두 탭을 합쳐 **한 번에 제출**합니다.
4. **우측 상단 "처음" 버튼** → 언제든 인트로 화면(또는 Master 화면)으로 돌아갈 수 있습니다.
5. **제출 완료** → 탭별 완료 현황 카드 + 응답 내용 `.txt` 다운로드.

### Master 계정

Master 비밀번호로 로그인하면 일반 설문 화면 대신 **업체별 LCA 데이터 관리 화면**이 표시됩니다.

- KOTITI LCA 대상 6개 업체가 아코디언 목록으로 나열됩니다.
- 업체를 펼치면 그 업체가 마지막으로 제출한 LCA(LCI) 데이터를 서버에서 불러와 표에 채웁니다(처음 제출이면 빈 폼).
- 데이터를 대신 입력·수정한 뒤 **저장·제출** 버튼을 누르면 해당 업체 시트에 새 제출 이력으로 반영됩니다.
- Master 수정 제출은 첨부파일을 다시 올리지 않습니다. 기존 첨부 링크는 이전 제출 이력의 행에 남아있고, Master가 남긴 최신 행에는 첨부 컬럼이 비어 있을 수 있습니다.
- Master 비밀번호는 `apps-script/Code.gs`의 `MASTER_PASSWORD` 상수입니다. **배포 전 반드시 강력한 값으로 교체**하세요.

## GAS 웹앱 배포 방법 (1회만 필요)

### 1단계. 새 스프레드시트 만들기

1. [Google Sheets](https://sheets.new) 에서 새 스프레드시트를 만드세요.
2. URL에서 `/d/` 와 `/edit` 사이의 ID를 복사합니다.  
   예: `https://docs.google.com/spreadsheets/d/**<여기가 ID>**/edit`

### 2단계. Apps Script 코드 붙여넣기

1. 스프레드시트에서 **확장 프로그램 → Apps Script** 를 엽니다.
2. 기존 코드를 모두 지우고 `apps-script/Code.gs` 전체를 붙여넣습니다.
3. 상단의 `const SPREADSHEET_ID = "..."` 를 1단계에서 복사한 ID로 교체합니다.
4. `const MASTER_PASSWORD = "master-2026";` 를 원하는 마스터 비밀번호로 교체합니다. **평문 이름(회사명)인 일반 기관 비밀번호와 겹치지 않는 값**을 쓰세요.
5. 저장(Ctrl+S / Cmd+S).

### 3단계. 웹앱으로 배포

1. 오른쪽 위 **배포 → 새 배포** 클릭.
2. 유형: **웹앱** 선택.
3. 설정:
   - 실행 계정: **나(me)**
   - 액세스 권한: **모든 사람(Anyone)**
4. **배포** 클릭 → Google 계정 권한 승인.
5. 완료 화면에서 **웹앱 URL** (`.../exec` 로 끝나는 URL) 복사.

### 4단계. index.html 에 URL 연결

`index.html` 에서 아래 줄을 찾아:

```js
const SHEET_ENDPOINT = "";
```

복사한 URL로 교체:

```js
const SHEET_ENDPOINT = "https://script.google.com/macros/s/.../exec";
```

저장 후 `main` 브랜치에 푸시하면 자동 반영됩니다.

## 시트 구조

GAS가 **기관별로 별도 탭(시트)**을 자동 생성합니다. 탭 이름은 기관명 그대로입니다.

- 헤더는 3행 계층 구조(섹션 / 서브섹션 / 항목명)로 자동 작성되고, 고정 열(제출일시·담당자 성명·부서·직위·연락처·이메일)과 마지막 `RAW_JSON` 열은 세로 병합됩니다.
- `RAW_JSON` 열에는 그 제출의 전체 payload(JSON)가 각 제출의 첫 행에만 저장됩니다. Master 계정의 `getLca` 조회는 이 값을 읽어 마지막 제출 데이터를 복원합니다.
- 제출 1건 = 1행(또는 KOTITI LCA 대상 기관의 경우, LCI 표 행 수만큼 여러 행 + 세로 병합). 제출할 때마다 새 행이 **추가(append)**되며 기존 행은 덮어쓰지 않습니다 — 즉 기관별 탭에는 전체 제출 이력이 누적됩니다.
- KOTITI LCA 대상 6개 기관(`ORG_HYEONHWANG_LIST`)은 LCI 표 데이터가 `lciTables` 필드로 개별 컬럼 방식 전송되어 시트에 펼쳐집니다. 그 외 기관은 `items` 배열의 라벨/값 쌍으로 저장됩니다.

⚠️ **마이그레이션 주의**: 이번 업데이트로 수출협회 PCF현황표 문항이 제거되어 제출 payload의 컬럼 구성이 바뀌었습니다.
이미 제출 이력이 있는 기관 탭은 기존 헤더와 컬럼 수가 어긋날 수 있으니, 배포 전 해당 기관 탭을 삭제(또는 헤더 3행만 지워 재생성)하는 것을 권장합니다.
